# Building MeCab for Windows x64

MeCab is used by the INTRP program to convert Japanese kanji to hiragana for display on the CRT. This is a one-time build step, similar to building CTranslate2.

The official MeCab Windows installer is 32-bit and from 2013 -- it cannot be used with the 64-bit MSVC build. You must build from source.

## Quick start (automated)

Run the build script from a command prompt:

```cmd
cd C:\git\williams_tube\yip_os
build_mecab_win.bat
```

This clones, patches, builds, compiles the dictionary, and installs to `C:\mecab_install`. You must apply the three source patches below **before the first build** (the script does not patch automatically).

## Prerequisites

- Visual Studio Build Tools (same as the main YipOS build)
- Git

## Source patches for modern MSVC

The MeCab source (last updated 2013) needs three patches to compile with modern MSVC (VS 2022+/C++17):

### Patch 1: `common.h` -- WPATH_FORCE undefined on MSVC

Upstream only defines `WPATH_FORCE` for `__GNUC__` on Windows, leaving MSVC with an undefined macro. Add the definition for MSVC:

```diff
 #ifdef _WIN32
+#define WPATH_FORCE(path) (MeCab::Utf8ToWide(path).c_str())
 #ifdef __GNUC__
-#define WPATH_FORCE(path) (MeCab::Utf8ToWide(path).c_str())
 #define WPATH(path) (path)
 #else
 #define WPATH(path) WPATH_FORCE(path)
 #endif
```

### Patch 2: `dictionary.cpp` -- std::binary_function removed in C++17

```diff
-struct pair_1st_cmp: public std::binary_function<bool, T1, T2> {
+struct pair_1st_cmp {
```

### Patch 3: `writer.cpp` -- ambiguous operator<< for size_t on x64

```diff
-          case 'L': *os << lattice->size(); break;
+          case 'L': *os << static_cast<unsigned long>(lattice->size()); break;
```

## Manual build steps

If you prefer to build manually instead of using `build_mecab_win.bat`:

### Step 1: Clone MeCab

```cmd
cd C:\
git clone https://github.com/taku910/mecab.git
cd mecab\mecab\src
```

### Step 2: Apply patches and generate config.h

Apply the three patches above, then create `config.h` in the `src` directory with:

```c
#ifndef MECAB_CONFIG_H_
#define MECAB_CONFIG_H_
#define HAVE_WINDOWS_H 1
#define HAVE_GETENV 1
#define HAVE_STRING_H 1
#define HAVE_STDLIB_H 1
#define HAVE_STDINT_H 1
#define HAVE_IO_H 1
#define HAVE_UNSIGNED_LONG_LONG_INT 1
#define DIC_VERSION 102
#define VERSION "0.996"
#define PACKAGE "mecab"
#define MECAB_DEFAULT_RC "mecabrc"
#define SIZEOF_CHAR 1
#define SIZEOF_SHORT 2
#define SIZEOF_INT 4
#define SIZEOF_LONG 4
#define SIZEOF_LONG_LONG 8
#define SIZEOF_SIZE_T 8
#endif
```

### Step 3: Build libmecab.dll (x64)

Open a **x64 Native Tools Command Prompt** (or run `vcvarsall.bat x64` first).

```cmd
cl /EHsc /O2 /MD /DDLL_EXPORT /DMECAB_USE_UTF8_ONLY /DHAVE_CONFIG_H ^
   /DUNICODE /D_UNICODE /D_CRT_SECURE_NO_DEPRECATE ^
   /I. /LD /Fe:libmecab.dll ^
   char_property.cpp connector.cpp context_id.cpp dictionary.cpp ^
   dictionary_compiler.cpp dictionary_generator.cpp dictionary_rewriter.cpp ^
   eval.cpp feature_index.cpp iconv_utils.cpp lbfgs.cpp learner.cpp ^
   learner_tagger.cpp libmecab.cpp nbest_generator.cpp param.cpp ^
   string_buffer.cpp tagger.cpp tokenizer.cpp utils.cpp viterbi.cpp ^
   writer.cpp ^
   /link advapi32.lib
```

### Step 4: Build mecab-dict-index.exe

```cmd
cl /EHsc /O2 /MD /DMECAB_USE_UTF8_ONLY /DHAVE_CONFIG_H ^
   /DUNICODE /D_UNICODE /D_CRT_SECURE_NO_DEPRECATE ^
   /I. /Fe:mecab-dict-index.exe ^
   dictionary_compiler.cpp dictionary_generator.cpp dictionary_rewriter.cpp ^
   char_property.cpp connector.cpp context_id.cpp eval.cpp ^
   feature_index.cpp iconv_utils.cpp lbfgs.cpp param.cpp ^
   string_buffer.cpp tagger.cpp tokenizer.cpp utils.cpp viterbi.cpp ^
   writer.cpp nbest_generator.cpp learner.cpp learner_tagger.cpp ^
   dictionary.cpp libmecab.cpp mecab-dict-index.cpp ^
   /link advapi32.lib
```

### Step 5: Compile the ipadic dictionary

```cmd
cd C:\mecab\mecab-ipadic
mkdir compiled
C:\mecab\mecab\src\mecab-dict-index.exe -d . -o compiled -f euc-jp -t utf-8
REM dicrc is NOT generated by mecab-dict-index — copy it manually
copy dicrc compiled\
```

### Step 6: Install to prefix

```cmd
mkdir C:\mecab_install\bin
mkdir C:\mecab_install\lib
mkdir C:\mecab_install\include
mkdir C:\mecab_install\dic\ipadic

copy C:\mecab\mecab\src\libmecab.dll C:\mecab_install\bin\
copy C:\mecab\mecab\src\libmecab.lib C:\mecab_install\lib\
copy C:\mecab\mecab\src\mecab.h C:\mecab_install\include\
xcopy /Y /Q C:\mecab\mecab-ipadic\compiled\*.* C:\mecab_install\dic\ipadic\
```

## Verify

Run `build_win.bat` from the `yip_os` directory. You should see:

```
MeCab found at C:\mecab_install
...
-- MeCab found -- Japanese kanji->hiragana enabled
```

At runtime, the Log tab should show:

```
MeCab: using dictionary at <exe_dir>\mecab-dic\ipadic
MeCab: Initialized successfully
```

## Troubleshooting

**`mecab-dict-index` fails with charset errors:**
Make sure you pass `-f euc-jp -t utf-8`. The ipadic source files are in EUC-JP encoding.

**CMake says "MeCab not found":**
Check that `C:\mecab_install\lib\libmecab.lib` and `C:\mecab_install\include\mecab.h` exist. The file names must match what CMakeLists.txt looks for: `find_library(MECAB_LIB mecab libmecab)` and `find_path(MECAB_INC mecab.h)`.

**Runtime "Failed to initialize" error:**
Check that `mecab-dic\ipadic\` exists next to `yip_os.exe` and contains `sys.dic`. The dictionary path is logged at startup.
